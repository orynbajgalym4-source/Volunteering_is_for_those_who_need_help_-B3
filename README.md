# Asar Telegram Mini App

Asar — система операционной готовности локальной помощи. Она превращает сообщения вроде «постараюсь прийти» в конкретные роли, подтверждения и видимый риск срыва.

## Один пользователь и одна боль

Основной пользователь — инициатор локального асара. Ему трудно понять, собраны ли действительно необходимые люди и ресурсы: обещания теряются в чатах, а критический отказ обнаруживается слишком поздно.

Asar не гарантирует явку. Он делает срыв заранее видимым и даёт инициатору время найти замену.

## Реализованный сценарий

1. Инициатор открывает Mini App через Telegram-бота; сервер проверяет подписанный `initData`.
2. Выбирает постоянный круг или создаёт новый с названием, описанием и фотографией.
3. Создаёт асар внутри круга и добавляет людей/ресурсы.
4. Отмечает критические опоры и публикует набор.
5. Отправляет общую гостевую ссылку для любого мессенджера.
6. Гость атомарно занимает свободное место и появляется среди участников круга.
7. По личной секретной ссылке подтверждает или отменяет участие.
8. Готовность пересчитывается как `NOT_READY → PROVISIONAL → READY`.
9. Отмена критической роли немедленно возвращает асар в `NOT_READY` и отправляет инициатору уведомление.
10. В день асара инициатор отмечает прибытие/неявку и фиксирует результат.

## Архитектура

- React 19 + TypeScript + Vinext/Vite;
- Cloudflare Worker‑совместимый серверный рендеринг;
- D1 (SQLite) для кругов, участников и асаров;
- R2 для фотографий кругов;
- route handlers для organizer и guest API;
- Telegram Mini Apps SDK и серверная HMAC-проверка `initData`;
- Bot API webhook, меню Mini App и нативное распространение приглашений;
- CSS variables Telegram и адаптация под safe area мобильных клиентов.

Сущности: `groups`, `group_members`, `asars`, `requirements`, `invites`, `commitments`. Сырые токены не используются как ключ поиска: в D1 хранится SHA‑256 hash. Точный адрес не возвращается публичному гостю и открывается участнику только после подтверждения.

## Локальный запуск

Требуется Node.js 22.13+.

```bash
npm install
npm run dev
```

Откройте `http://localhost:3000`. Локально organizer API использует профиль `telegram:demo`; в опубликованной версии отсутствие валидного Telegram `initData` возвращает `401`.

Для опубликованной версии нужны runtime-переменные:

```env
TELEGRAM_BOT_TOKEN=
TELEGRAM_BOT_USERNAME=
TELEGRAM_WEBHOOK_SECRET=
```

Токен и секрет webhook должны храниться только как секреты среды, не в репозитории.

## Миграции

```bash
npm run db:generate
```

SQL сохраняется в `drizzle/` и упаковывается вместе с Sites build. Route handlers также безопасно создают отсутствующие таблицы через `CREATE TABLE IF NOT EXISTS`, чтобы локальный первый запуск не требовал ручной подготовки.

## Проверка

```bash
npm test
npm run lint
```

Тесты покрывают критический расчёт готовности, отмену, запрещённые переходы жизненного цикла и проверку подписи Telegram `initData`.

## Приватность

Продукт не собирает диагнозы, документы, финансовые сведения и семейную историю получателя. Публичная ссылка показывает примерный район; точный адрес доступен только подтверждённому участнику. Контакт участника видит только инициатор.

## Демо за 90 секунд

Откройте Mini App из меню бота, создайте асар с водителем и машиной и отправьте приглашение в другой Telegram-чат. Два гостя занимают и подтверждают роли — статус становится `READY`. Затем водитель отменяет участие: инициатор получает предупреждение и кнопку «Найти замену», а панель показывает `NOT_READY`.

## Ограничения MVP

Нет внутреннего чата, донатов, рейтинга надёжности, глобальной ленты и карты волонтёров. Резервные участники, плановые напоминания и загрузка итогового фото оставлены после основного сценария.
