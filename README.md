# Asar MVP v1.0

Asar — система операционной готовности локальной помощи. Она превращает сообщения вроде «постараюсь прийти» в конкретные роли, подтверждения и видимый риск срыва.

## Один пользователь и одна боль

Основной пользователь — инициатор локального асара. Ему трудно понять, собраны ли действительно необходимые люди и ресурсы: обещания теряются в чатах, а критический отказ обнаруживается слишком поздно.

Asar не гарантирует явку. Он делает срыв заранее видимым и даёт инициатору время найти замену.

## Реализованный сценарий

1. Инициатор входит через Sign in with ChatGPT (локально используется демо‑профиль).
2. Создаёт асар в три шага и добавляет людей/ресурсы.
3. Отмечает критические опоры и публикует набор.
4. Создаёт общую гостевую ссылку либо ссылку только на одну нехватку.
5. Гость без аккаунта атомарно занимает свободное место.
6. По личной секретной ссылке подтверждает или отменяет участие.
7. Готовность пересчитывается как `NOT_READY → PROVISIONAL → READY`.
8. Отмена критической роли немедленно возвращает асар в `NOT_READY`.
9. В день асара инициатор отмечает прибытие/неявку и фиксирует результат.

## Архитектура

- React 19 + TypeScript + Vinext/Vite;
- Cloudflare Worker‑совместимый серверный рендеринг;
- D1 (SQLite) для постоянных данных;
- route handlers для organizer и guest API;
- Sign in with ChatGPT для идентичности инициатора;
- CSS variables и адаптивная мобильная система.

Сущности: `asars`, `requirements`, `invites`, `commitments`. Сырые токены не используются как ключ поиска: в D1 хранится SHA‑256 hash. Точный адрес не возвращается публичному гостю и открывается участнику только после подтверждения.

## Локальный запуск

Требуется Node.js 22.13+.

```bash
npm install
npm run dev
```

Откройте `http://localhost:3000`. Локально organizer API использует профиль `demo@asar.local`; в опубликованной версии отсутствие удостоверенной личности возвращает `401`.

## Миграции

```bash
npm run db:generate
```

SQL сохраняется в `drizzle/` и упаковывается вместе с Sites build. Route handlers также безопасно создают отсутствующие таблицы через `CREATE TABLE IF NOT EXISTS`, чтобы локальный первый запуск не требовал ручной подготовки.

## Проверка

```bash
npm test
npm run lint
```

Тесты покрывают критический расчёт готовности, отмену и запрещённые переходы жизненного цикла.

## Приватность

Продукт не собирает диагнозы, документы, финансовые сведения и семейную историю получателя. Публичная ссылка показывает примерный район; точный адрес доступен только подтверждённому участнику. Контакт участника видит только инициатор.

## Демо за 90 секунд

Создайте асар с водителем и машиной, опубликуйте его, откройте общую ссылку в приватном окне и займите обе роли. Подтвердите обе личные ссылки — статус станет `READY`. Затем отмените водителя: панель сразу покажет `NOT_READY` и предложит создать ссылку только на водителя. Новый гость занимает роль и восстанавливает готовность.

## Ограничения MVP

Нет внутреннего чата, донатов, рейтинга надёжности, глобальной ленты, карты волонтёров и автоматических Telegram‑уведомлений. Резервные участники и загрузка итогового фото оставлены после основного сценария.

## AI‑инструменты

AI использовался для ускорения реализации и проверки спецификации. Доменные правила, приватность, пользовательские состояния и итоговый код проверяются тестами и сборкой проекта.
